## 🔄 Daily Project Generation Process

### Morgens (8:00) — Morning Briefing + Projekt-Ideen

**Generator läuft autonom:**

1. **Trend-Analyse** (Web-Search)
   - Product Hunt Trending
   - GitHub Trending
   - Twitter/X Tech-Trends
   - HackerNews Top
   - Reddit r/startups

2. **Competitor-Scan**
   - Neue Features bei Wettbewerbern
   - Pricing-Changes
   - User-Complaints (Twitter, Reddit)
   - Feature-Gaps identifizieren

3. **Ideen-Generierung** (AI)
   - 5-10 neue Projekt-Ideen
   - Nach Kategorien sortiert
   - Mit Priorität (Impact/Effort)
   - Mit Tech-Stack-Vorschlag

4. **Output im Morning Briefing:**
   ```
   🚀 PROJEKT-IDEEN DES TAGES

   🔴 HIGH PRIORITY (diese Woche startbar)
   1. [Name] — [1-Satz Beschreibung]
      - Markt: [Größe]
      - Revenue: [Potenzial]
      - Tech: [Stack]
      - MVP: [Tage]
   
   🟡 MEDIUM PRIORITY (nächste 2 Wochen)
   2. [Name] — [Beschreibung]
   
   🟢 LOW PRIORITY (Backlog)
   3. [Name] — [Beschreibung]
   ```

---

### Nachmittags (14:00) — Projekt-Update + Neue Ideen

**Generator läuft autonom:**

1. **Morning-Ideen Review**
   - User-Feedback gesammelt?
   - Tech-Research gemacht?
   - Priorität angepasst?

2. **Neue Ideen generieren** (wie morgens)
   - Weitere 5-10 Ideen
   - Andere Kategorien priorisieren

3. **Autonome Entscheidung:**
   - Top-Idee auswählen
   - Sub-Agent spawnen für MVP
   - Oder: Research vertiefen

4. **Output im Afternoon Update:**
   ```
   📈 PROJEKT-UPDATE

   ✅ Morning-Ideen Status:
   - Idee 1: Research läuft
   - Idee 2: MVP gestartet
   - Idee 3: Backlog
   
   🆕 NEUE IDEEN (Nachmittag)
   1. [Name] — [Beschreibung]
   2. [Name] — [Beschreibung]
   
   🎯 NÄCHSTE AKTION:
   - [Projekt X] wird heute gestartet
   ```

## 🤖 Autonome Projekt-Entwicklung

### Wenn Idee ausgewählt wird:

**Phase 1: Research (2-4h)**
- Sub-Agent spawnen
- Markt-Analyse
- Competitor-Research
- Tech-Stack entscheiden
- Feature-Liste erstellen

**Phase 2: MVP (1-3 Tage)**
- Sub-Agent spawnen (oder mehrere parallel)
- Core-Features bauen
- Basic UI/UX
- Deploy-ready

**Phase 3: Launch (1 Tag)**
- README schreiben
- Landing Page
- Social Media Posts
- Launch auf Product Hunt / GitHub

**Phase 4: Optimization (laufend)**
- Auto-Optimization-Jobs aktivieren
- User-Feedback sammeln
- Iterative Verbesserungen
- Feature-Erweiterungen

## 📋 Projekt-Bewertungsmatrix

### scoring (0-100 Punkte)

| Faktor | Gewicht | Bewertung |
|--------|---------|-----------|
| **Markt-Größe** | 20% | TAM, SAM, SOM |
| **Revenue-Potenzial** | 20% | MRR möglich |
| **Wettbewerb** | 10% | Weniger = besser |
| **Tech-Komplexität** | 10% | Einfacher = besser |
| **Time-to-MVP** | 15% | Schneller = besser |
| **Viral-Potenzial** | 10% | Word-of-Mouth |
| **Strategic Fit** | 15% | Passt zu Portfolio |

### Prioritäts-Grenzen:

- **🔴 HIGH:** Score ≥ 75 → Diese Woche starten
- **🟡 MEDIUM:** Score 50-74 → Nächste 2 Wochen
- **🟢 LOW:** Score < 50 → Backlog
